## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller content -> store/import redaction | manual & import text may carry secrets; redaction is the last gate before persistence |
| stored rows -> redactExisting UPDATE | pre-existing rows are rewritten in place |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-09 | Information Disclosure | manual/import write paths bypassing redaction | mitigate | Task 1 routes both paths through applyRedaction before persist; tests assert raw secrets absent in stored content (closes the CAPT-04 bypass noted in D-06) |
| T-06-10 | Information Disclosure | redactExisting preview echoing secrets | mitigate | Previews are length-truncated and built from redacted/contextual text per D-07/D-14; acceptance asserts bounded preview length |
| T-06-11 | Tampering | updateMemoryContent SQL | mitigate | Parameterized projectId/memoryId/content binding mirrors updateMemoryImportance; no interpolation |
| T-06-12 | Repudiation | silent redaction failure | mitigate | redaction_partial_failure warning code surfaced in store/import response envelopes (D-08) so callers can observe partial failure |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs introduced by this plan; if any added, gate via legitimacy checkpoint |
