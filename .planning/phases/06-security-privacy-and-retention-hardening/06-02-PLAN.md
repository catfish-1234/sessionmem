<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/adapter request -> pruneMemories | retentionDays and projectId are caller-supplied and drive a DELETE |
| service -> SQLite | DELETE affects persistent rows irreversibly (hard-delete) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering | deleteMemoriesOlderThan SQL | mitigate | Parameterized projectId + cutoff binding; acceptance criterion asserts placeholders, no string interpolation — prevents SQL injection of the cutoff/project filter |
| T-06-06 | Denial of Service / data loss | wrong-rows deletion | mitigate | dryRun defaults true (D-12); cutoff scoped by project_id AND created_at; integration test proves only intended rows deleted and session_events/survivors retained |
| T-06-07 | Repudiation | unbounded delete on misconfig | mitigate | retentionDays<=0 disables pruning entirely (returns 0); negative/zero cannot translate into a future cutoff that deletes everything |
| T-06-08 | Information Disclosure | error leakage | accept | Errors route through existing toErrorEnvelope which already shapes DomainError codes/messages; no new sensitive data exposed |
</threat_model>
