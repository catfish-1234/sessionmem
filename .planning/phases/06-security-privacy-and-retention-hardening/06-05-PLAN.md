<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args/flags -> retention prune | operator-supplied --force/--days drive a hard-delete |
| CLI args -> config set | operator writes keys/values into config.json consumed as policy |
| install -> config.json | install seeds policy defaults into the user filesystem |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-16 | Denial of Service / data loss | retention prune deleting without confirmation | mitigate | Dry-run is default (D-12); deletion requires explicit --force; test asserts no deletion without --force |
| T-06-17 | Tampering | config set writing arbitrary/unknown keys | mitigate | configSet validates key against known config keys and coerces types; unknown key rejected with exit 1 and no file write (acceptance asserts file unchanged) |
| T-06-18 | Tampering | install clobbering user config | mitigate | install writes only when file absent (D-10); test asserts pre-existing retentionDays preserved |
| T-06-19 | Information Disclosure | error output | accept | CLI prints DomainError messages to stderr (existing Phase 5 convention); no secrets surfaced |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan (commander/zod already present); no legitimacy gate required |
</threat_model>
