<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> policyConfig | `~/.sessionmem/config.json` is operator-writable; contents are parsed into runtime policy |
| memory content -> redaction rules | session/manual/import text (potentially containing secrets) flows through regex rules |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | defaultRules() regex coverage | mitigate | Behavior tests assert each D-05 category redacts; placeholders replace raw secret so leakage is observable in test output |
| T-06-02 | Tampering | config.json parsing | mitigate | readPolicyConfig wraps JSON.parse + zod validation in try/catch and falls back to DEFAULT_POLICY_CONFIG; malformed/injected JSON cannot crash or set unvalidated keys |
| T-06-03 | Denial of Service | redaction regex (ReDoS) | mitigate | Keep patterns anchored with explicit prefixes (AKIA, ghp_, eyJ) and bounded quantifiers; avoid nested unbounded backtracking; prose-no-op test guards catastrophic cases |
| T-06-04 | Elevation of Privilege | unknown config keys | mitigate | zod schema with fixed shape rejects unknown keys; only retentionDays/redactionEnabled are honored |
</threat_model>
