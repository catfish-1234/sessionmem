<objective>
Deliver the core team-sync capability (TEAM-01). Add a dedicated `pullMemories` service method that merges teammate memory records via the `importMemories`-style `ON CONFLICT(id) DO UPDATE` upsert, but with three changes: importance-preservation (`MAX(memories.importance, excluded.importance)`, D-11), `author`/`origin_project_id` stamping (D-06), and the cross-project-id-collision skip preserved (D-09). Pull re-runs `applyRedaction` on every incoming record as a 4th write path (D-12). Then add a `sessionmem sync` command that pushes a full local snapshot to `{sharedPath}/{project_id}/{username}.json` (atomically) and pulls every other teammate file in that directory, printing the D-16 summary.

Purpose: This is the user-facing payoff of the phase — the actual push/pull/merge that lets teammates share memory through a shared filesystem path. It depends on Plan 01's columns/author and Plan 02's team config/username.

Output: `pullMemories` + `pullMemoriesRequestSchema`, a `sync.ts` command, registration in `index.ts`, a two-user test helper, and unit + integration specs.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Teammate JSON files → local DB | The primary new untrusted-input surface. Anyone with write access to {sharedPath} can place files. |
| Shared filesystem → app | OS-level ACLs on {sharedPath} are the real trust boundary; the app treats writers as trusted teammates. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02 | Information Disclosure | pull write path | mitigate | Re-run `applyRedaction` on every pulled record (D-12, 4th write path) regardless of the teammate's redaction setting; re-embed redacted text. |
| T-07-03 | Tampering / DoS | malformed/oversized teammate JSON | mitigate | Per-file `JSON.parse` try/catch skip-and-warn + array guard; per-record `importMemoryRecordSchema.safeParse` skip-and-warn — one bad file never aborts the pull (reuse import.ts hardening). |
| T-07-06 | Tampering | cross-project id collision | mitigate | Preserve the `ownerStmt` cross-project skip from importMemories in pullMemories (D-09) — a colliding id from another project is skipped, not overwritten. |
| T-07-07 | Tampering | path traversal via teammate filename/embedded projectId | mitigate | Enumerate files with `readdirSync` (no path injection); build write paths from local `projectId` + sanitized `username` via `path.join`; never use a path string from inside a teammate file. |
| T-07-11 | Spoofing | teammate sets arbitrary `author` in their JSON | accept | author is advisory provenance, not a security control; trust boundary is the shared-dir ACL. Documented in Plan 04 setup docs (RESEARCH Security Domain). |
| T-07-12 | DoS | partial-file read mid-push on a network drive | mitigate | Atomic write-to-temp-then-rename on push (Pitfall 4) + skip-and-warn parse on pull. |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase (RESEARCH Package Legitimacy Audit: none). |
</threat_model>
