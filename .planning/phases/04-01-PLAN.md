# Phase 4, Plan 1: Core Adapter Factory and Auto-Detect Logic

## Goal
Establish the core `src/adapters/` architecture to detect the host environment and initialize the right adapter behavior.

## Requirements Mapped
- Part of PLAT-08 (Generic MCP path)

## Proposed Changes
1. **`src/adapters/factory.ts`**
   - Create `AdapterFactory` to inspect environment variables, process names, and system state to detect the current host (Cursor, Windsurf, Claude Code, Antigravity, etc.).
   - If detection fails, fallback to a `GenericMCPAdapter`.
   - Ensure the factory resolves an `AdapterContract` interface.

2. **`src/adapters/contract.ts`**
   - Refine the existing `AdapterContract` to explicitly support gracefully missing capabilities (e.g., `supportsPrompts`, `supportsResources` flags).

## Verification
- Unit tests validating auto-detection rules against mocked environment variables.
