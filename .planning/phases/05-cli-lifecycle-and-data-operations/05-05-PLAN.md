<objective>
Close the single shared root-cause BLOCKER from 05-VERIFICATION.md (status: gaps_found, 4/9): `src/cli/index.ts` registers `.action(searchCommand)` (and list/show/stats/forget/export/import) with bare function references. Commander always appends its own `Command` instance as the final argument to action callbacks, so each command's trailing `ctx?: CliContext` parameter receives the `Command` object. `ctx ?? createCliContext()` then resolves to that `Command` object — which has no `.service` — and every one of these 7 commands crashes at runtime with `TypeError: Cannot read properties of undefined (reading 'call')`.

This bug was invisible to the existing test suite (135 passing tests) because every spec calls the command functions directly with an explicitly injected `CliContext`, never through commander's real dispatch.

Purpose: Restore CLI-03 (search), CLI-04 (list/show/forget), CLI-05 (export/import), CLI-06 (stats) to working state via the actual `sessionmem` binary, fix the secondary `--limit` string-vs-number defect for search, and add a regression test that exercises the real commander entrypoint so this class of bug is caught going forward.
Output: corrected src/cli/index.ts wiring, --limit coercion in search.ts, and a new end-to-end spec that spawns dist/cli/index.js.
</objective>
