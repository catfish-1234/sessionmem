# Phase 4, Plan 4: Missing Capability Fallbacks

## Goal
Ensure sessionmem functions gracefully on limited hosts lacking MCP resources or prompts.

## Requirements Mapped
- Implicit coverage for limited hosts (supports PLAT-08 reliability)

## Proposed Changes
1. **`src/adapters/capabilities/fallbackTools.ts`**
   - Implement a `fetch_memories` MCP Tool. If the host lacks `resources` support, memory retrieval is exposed entirely via this tool.
   - Implement a startup injection fallback tool. If the host lacks `prompts`, the agent is instructed (via system prompt or initial tool discovery) to run an initialization tool.

2. **`src/adapters/contract.ts` Updates**
   - Expose the fallback tools dynamically during MCP server initialization only if the host capabilities handshake indicates missing features.

3. **`src/core/session.ts` Updates**
   - Log warnings (but continue running) if connection drops mid-session.

## Verification
- Unit tests simulating restricted host capabilities handshake and validating that fallback tools are registered.
