# Phase 4, Plan 5: Manual Config and Test Commands

## Goal
Provide a reliable manual fallback experience when automatic injection fails.

## Requirements Mapped
- PLAT-08 (Generic MCP fallback UX)
- Setup for CLI operations (precursor to Phase 5)

## Proposed Changes
1. **`src/cli/commands/run.ts`**
   - Implement the `sessionmem run` command to launch the basic stdio MCP server directly.
   - Setup log output targeting `~/.sessionmem/logs/mcp.log` for debugging visibility.

2. **`src/cli/commands/install.ts` Updates**
   - If auto-config fails, fallback to printing an exact copy-paste JSON block for manual MCP configuration in the user's host of choice.

3. **`src/adapters/tools/ping.ts`**
   - Implement a `sessionmem_ping` tool.
   - This simple tool returns a static success string and version, allowing users to quickly verify their manual configuration works.

## Verification
- CLI integration test for the `run` command startup.
- Verification that `sessionmem_ping` is registered across all adapter configurations.
